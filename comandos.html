Las siguientes utilidades son utilizadas en la mayoría de sistemas operativos y permiten diagnosticar problemas, controlar los servicios y cambiar la configuración.

PING

Ping (Packet Internet Groper) es una utilidad que sirve para enviar mensajes a una dirección concreta de red con el fin de realizar una comprobación utilizado en el protocolo ICMP. El nodo destinatario nos responderá con un paquete para confirmarnos que ambos nodos pueden comunicarse perfectamente.

ifconfig / ip a

Permite obtener la información de la configuración IP del nodo, así como modificarla de forma no persistente. El comando iwconfig permite ver la configuración IP de las conexiones inalámbricas.

netstat

netstat proporciona información sobre las conexiones de red.

ip route

ip route muestra la tabla de direccionamiento del nodo en el que se ejecuta.

traceroute

traceroute nos muestra los routers por los que pasa un paquete hasta llegar a su destino.

nslookup

nslookup permite realizar consultas de nombres de dominio y comprobar qué servidor nos ofrece la respuesta.

ifup / ifdown

ambos comandos permiten activar o desactivar la interfaz de red indicada.

Para más información de cada utilidad puede utilizar el comando man seguido de la utilidad de la que se quiere mostrar información.

Por ejemplo: man nslookup

Todos estos comandos son de NetTools, no obstante una gran mayoría de sistemas operativos linux están optando por utilizar iproute2. Podemos encontrar equivalencias entre los comandos tradicionales de ifconfig y los comandos de iroute2 por ejemplo aquí

1.5.2. Gestión de servicios

Para la gestión de los servicios se utiliza la utilidad service de la siguiente forma:

# service nombre-del-servicio [stop|start|force-load|restart|status]

Es muy importante que una vez que configuramos un servicio y lo arrancamos, start, luego debemos ver si este está funcionando correctamente. Para ello utilizamos el comando status, el cual nos dará el estado de funcionamiento del mismo.

Disponemos además de systemctl con el que podemos también levantar, parar, etc. los servicios. Veamos una comparativa:

service systemctl Desctripción
service name start systemctl start name.service Inicia un servicio.
service name stop systemctl stop name.service Detiene un servicio.
service name restart systemctl restart name.service Reinicia un servicio.
service name cond restart systemctl try-restart name.service Reinicia un servicio sólo si se está ejecutando.
service name reload systemctl reload name.service Vuelve a cargar la configuración.
service name status

systemctl status name.service

systemctl is-active name.service

 Comprueba si un servicio se está ejecutando.
service --status-all systemctl list-units --type service --all Muestra el estado de todos los servicios.
